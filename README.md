> 「2026 年还在面 LeetCode 的公司不值得一去」——真的吗？
>
> 在 AI 时代，面试的目的正在改变。手撕算法到底还有没有意义？纯背题、纯手写还剩多少？哪些公司转向了**实战编码 / 系统设计 / AI 结对**？这个 repo 想给中文社区的开发者一个**透明、可交流、众包**的答案。

这是一个**开放贡献**的清单。欢迎你把自己的面试经历贡献进来——无论公司是否还在面 LeetCode。数据越多，社区越受益。

> ⚠️ **免责声明**：本仓库内容来自社区众包，反映的是个别面试者在特定时间、特定岗位的经历，**不代表公司官方政策，也不构成对任何公司的评价或求职建议**。同一公司不同部门 / 岗位 / 面试官差异很大。请理性参考，欢迎纠错。

## 🤔 为什么做这个 repo？

「还在面 LeetCode」常被当成一家公司**技术理念过时**的信号。但在 2026 年，这个判断变得复杂了：

- **AI 让纯背题失去意义**——LLM 几秒就能写出最优解，考「你背没背过」越来越没价值。
- **但算法思维仍然重要**——很多公司把 LC 当成**思维过滤器**，而非记忆测试，允许讨论、允许 AI 辅助。
- **面试形式在分化**——有的转向实战编码、系统设计、AI 结对编程；有的依然原样手撕。

与其争论「值不值得去」，不如**把事实摆出来**，让每个人根据自己的判断做选择。这个 repo 不下结论，只提供数据。

详细的趋势观察见 [docs/trends.md](docs/trends.md)。

## 🙌 如何贡献

三种方式，任选其一：

1. **直接改表格（推荐，最快）**——点 [README.md](README.md) 右上角 ✏️，在公司清单里加一行或更新一行，提 PR。格式见 [CONTRIBUTING.md](CONTRIBUTING.md)。
2. **开 Issue**——不想动 Markdown？用 [📝 提交面试经历](../../issues/new?template=submit-interview.yml) 模板填一下，维护者帮你录入。
3. **参与讨论**——对「AI 时代 LC 还有没有意义」有想法？欢迎到 [Discussions](../../discussions) 聊。

**贡献准则**：只填你**亲历或可靠来源**的信息；不带攻击性、不泄露具体面试题；尊重 NDA。详见 [CONTRIBUTING.md](CONTRIBUTING.md)。
