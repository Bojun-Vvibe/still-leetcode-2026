# 贡献指南

感谢你愿意为社区贡献一手信息！🎉 这份清单的价值完全来自大家真实的面试经历。

## 核心原则

1. **只填亲历或可靠来源的信息**——你自己的面试、朋友的一手转述、可信的公开面经。道听途说请标注来源为「传闻」。
2. **不泄露具体面试题**——尊重 NDA。描述「形式」「难度」「方向」即可，不要贴原题。
3. **客观中立**——这是事实清单，不是评价 / 吐槽区。不带人身攻击，不引战。
4. **保护隐私**——不要写出具体面试官姓名或可识别个人的信息。

## 如何添加 / 更新一家公司

直接编辑 `README.md` 里「📊 公司清单」的表格，加一行或更新已有行。请尽量按**公司拼音 / 字母顺序**插入。

### 字段说明

| 字段 | 填法 | 说明 |
|------|------|------|
| **公司** | 公司名 | 可加部门，如 `字节·抖音`。规模较小的初创可注明 `(初创)`。 |
| **是否面 LC** | `✅` / `🟡` / `❌` / `❓` | ✅ 仍以 LC 算法为主；🟡 部分或看岗位；❌ 基本不考纯算法；❓ 数据不足。 |
| **面试形式** | 简短描述 | 如 `现场手写`、`OA + 现场`、`系统设计`、`实战项目`、`AI 结对`。 |
| **难度** | `易` / `中` / `难` / 区间 | 如 `中-难`。不适用填 `—`。 |
| **AI 工具政策** | 简短描述 | 面试中是否允许用 Copilot / ChatGPT / 查文档。未知填 `未知`。这是 2026 年的关键维度。 |
| **信息来源** | 来源类型 | 如 `亲历`、`面经`、`内推`、`朋友面试`、`社区`、`传闻`。 |
| **更新时间** | `YYYY-MM` | 你这次信息对应的时间点。面试情况会变，时间很重要。 |

### 示例

```markdown
| 字节跳动 | ✅ | 现场手写 + OA | 中-难 | 面试中禁用 AI | 亲历 | 2026-05 |
```

### 已有公司怎么更新？

- 信息**变化**了（比如政策改了）：直接更新那一行，并把「更新时间」改成最新。
- 你有**不同经历**：如果和现有记录冲突，可以把「是否面 LC」改成 `🟡`，并在面试形式里简注差异（如 `看岗位/面试官`）。众包数据有分歧很正常，如实反映即可。

## 提交流程

1. Fork 本仓库（或直接在 GitHub 网页点 README 的 ✏️ 编辑，会自动 fork）。
2. 修改 `README.md`。
3. 提交 Pull Request，标题写清公司名，如 `添加：美团（2026-06）`。
4. PR 描述里**简单说明信息来源**（不需要暴露隐私），方便维护者判断。

不会用 Git？没关系——用 [📝 提交面试经历 Issue 模板](../../issues/new?template=submit-interview.yml) 填一下，维护者会帮你录入。

## 自动校验

提 PR 后，GitHub Action 会自动检查表格格式（列数、`✅/🟡/❌/❓` 标记、`YYYY-MM` 时间等）。你也可以在本地先跑一遍：

```bash
python3 scripts/validate_table.py
```

绿勾通过即可合并；红叉会指出具体哪一行有问题。
